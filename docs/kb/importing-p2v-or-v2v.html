Verge.io MOVE Tool

Clone ISO Transfer Utility

Mar 2021

ISO Version 4.8.2

For PtoV and VtoV Import

 

The Verge.io MOVE tool (Verge.io-clone.iso) can be used to import external servers (physical or virtual) into Verge.io virtual machines.  Zero block detection is used in transferring source drives, providing efficiency as only used blocks require transfer.  The Verge.io Clone Utility is recommended for PtoV imports (importing physical computers) and for non-VMware VtoV (importing existing virtual machines). The verge VMware connector is recommended for importing VMware virtual machines.

  

Prerequisites

Verge System (destination):

  • Network URL that can be accessible from source VMs to be imported.
  • Verge.io username/password with permissions to create VMs
  • Adequate available vSAN storage for cloned VM drives (Note: Verge.io vSAN global deduplication handles zero blocks, thus only actual used drive space required.) 
  • Although not necessary for the clone-import process itself, adequate resources (RAM/CPU) will need to be available on the destination verge system to run imported VMs.

VM/Physical Machine to Clone (source): 

  • Network connection (attached to network that can reach the destination verge system)
  • Attached USB/CD-ROM device (containing Verge.io-clone.iso)
  • Minimum 1GB RAM (to boot from/run the iso)

 

Determining Appropriate Sync Method

The appropriate sync method to use will depend on the nature of the network connection between the source VM and destination Verge.io system. 

HTTPS upload

Transferring using this method has minimal network configuration requirements.  This transfer method is automatically selected by default. 

vSAN-direct Sync

The vSAN-direct sync method can be used for optimum throughput over a LAN connection; it is not intended for WAN connections. 

vSAN-direct syncs are only allowed to the host system (not directly to a tenant) 

This method provides for faster transfer speeds, but involves additional network requirements.    The vSAN-direct method can be selected under Advanced settings from the confirmation screen (last dialog). 

Additional Network Requirements for vSAN-direct Sync method:

  • Source side static IP
  • Destination Verge.io system static IP
  • NAT and SNAT rules on the UI(external) and core networks of Verge.io system

Refer  to  Direct vSAN Network Configuration section for details.  

 

Obtaining the Verge.io-clone.iso File

  1. From the Main Dashboard, click Backup/DR on the left menu.
  2. Click the Download ISO button in the Clone ISO Utility section. 
  1. The Clone ISO dialog appears giving options for creation of a public download link for the ISO file: 
  • No (default selection)- no public download link created.  The Clone ISo will only be downloadable from the verge UI.
  • Yes, make the link Anonymous (uuid) - creates a public download link with GUID (128bit long number) 

(ex: https://verge.example.com/273b5d6a--b205-1e1b-6ae6-01cfa7ed1233)

  • Yes, use filename as the link - creates a public download link using the original file name (ex: https://verge.example.com/verge.io-clone.iso)
  • Yes, specify a custom name - creates a public download link with file name entered (ex: https://verge.example.com/myfilename.iso)
  1. Click OK after selecting the desired option.  The build process may take a few minutes to complete.  
  1. Once the build is complete, the Clone ISO section will show a Status of Ready.  Additionally the version, public link (if applicable) and the build date/time will display.    Note: the copy icon next to the public link field can be used to duplicate the URL to the clipboard.  

 

  1. Download the ISO; it can now be downloaded from: 
  • within the verge UI

-OR-

  • the public link (if created)

The bootable Clone ISO can be used on CD or USB.

 

Using the Verge Clone Utility 

  1. Power down the source computer and boot from the Clone ISO ( Verge.io-clone.iso) 
  2. Hit <Enter> or simply allow the clone utility to boot automatically. 
  1. Select Launch Clone Utility.
  1. Hit <Enter> to confirm the introduction message. 
  1. Select the NIC to be used to connect to the destination Verge.io system.  Only one NIC is selected for connecting to the destination; however, all NICS from the source system are brought over in the VM import. 
  1. Select appropriate choice (DHCP or Static) for the connecting network. 

 

Static IP configuration

 If static IP configuration selected, use up/down ↑/↓  arrows to select different fields.  Selected field is highlighted in blue.   Hit <Enter> while a field is selected to modify that field.  Once all the static fields are configured as needed, use arrow keys to highlight the Done option and <Enter> to proceed.  

  1. Enter the URL (hostname or IP address) of the verge system.

 

  1. Enter verge username and password. Note: user must have permissions to create and modify virtual machines to successfully complete a clone import.
  1. If the utility is successful in connecting to the verge system, a success message displays.

Resuming a Clone import

A Clone import that successfully started, but did not complete (due to a network disruption, for example) can be resumed.   

  1. Boot the source computer from the verge clone iso again and follow the same process, being sure to input the VM name exactly as it was the first time.  (The VM name can be verified on the destination verge system, in the Virtual Machines list.)
  1. Use the right arrow → key or <Tab> to select the Resume option, and hit <Enter>.

The clone import will continue from where it left off. 

Rebuilding the Clone ISO

The Clone ISO can be rebuilt in order to take advantage of code updates.  

To manually initiate this process: 

  1. From the Main Dashboard, click Backup/DR on the left menu.
  2. Click Edit Clone ISO on the left menu. 
  3. Check the Force Rebuild checkbox.
  4. Click OK.

The rebuild may take a few minutes.  While in process, status will indicate “Building”

Status will change to “Ready” when the ISO build is complete. 

 

Rebuilding the ISO can be automated.  Some customers opt to update the clone ISO anytime the system is updated.  

To schedule automatic ISO rebuilds:

  1. From the main dashboard, click System on the left menu.
  2. Click Tasks/Events on the left menu. 
  3. Click New on the left menu. 
  4. Configure the task as shown to automatically rebuild the ISO each time the verge system is updated :
  1. Click Submit to save the new task.

 

Clone Utility Troubleshooting

ISSUE: Failed dhcp

DHCP option was selected but no dhcp service found

  • Verify there is a dhcp service available via the selected NIC.
  • Verify the selected network actually has a connection.  

ISSUE: Login failed message

  • Verify input of correct username and password 
  • The Login failed message can also appear when there is an issue with network connection or URL.  Verify input of correct Verge.io system host/IP address. Note: the address should be entered without preceding “https://” 

 

  • Verify static IP configuration information (if applicable)
    • The IP address and gateway must be valid addresses for the attached network. Note: IP address must be entered in cidr notation (e.g. 192.168.0.50/24).
    • Verify no IP conflict with the used IP address.
    • Correct DNS address must be used to connect the verge system via hostname.
  • Verify underlying network connection.  Boot the computer without the clone ISO to test reaching the Verge.io host/IP address; confirm connection over specific NIC, using same configuration (e.g. dhcp or static addressing).

ISSUE: OPENSSL errors during the transfer

Openssl errors during the transfer indicate a network issue.   The clone operation will retry and can often recover from an openssl error.   The clone operation will abort if timeout or number of retries threshold reached.

  • A possible cause for sporadic OPENSSL errors, can be MTU size mismatches.  Verify MTU configuration on connecting networks.  

Note: 

If a network issue causes a clone import to fail before finishing: a partial clone import can be restarted where it left off by:

  •  resetting the machine and booting again from the verge Clone ISO

-or- 

  •  using the <ESC> key to return the command line and use the command: 

/usr/bin/verge.io-clone-vm.sh

See directions above for Resuming a Clone import 

ISSUE: Permission denied

If the verge user provided does not have adequate permissions, the clone will begin and create the new Virtual Machine record; however, it will not be able to create any VM drives or NICs.

  • Restart the clone ISO providing username/password of a verge user with List, Read, Create and Modify permissions to Virtual Machines.  Note: Since the empty VM will have been created on the destination verge system (when run initially without adequate permissions): when run again with proper permissions, the clone operation can be performed as a resume or the VM can be removed on the destination system before starting the clone again.  

Direct vSAN Network Configuration

Direct vSAN transfer method is provided for use over local networks; it is not intended for use over WAN connections.

Direct vSAN transfer method can only be used to transfer to a root system (not directly to a Tenant).

In order to use the vSAN Mount (direct sync) method of the Clone Iso utility, three networking rules must be in place (2 on the core network and 1 on the external ui network).

The following section outlines the network rules necessary for a system to accept a Clone ISO transfer using the direct vSAN method.  For a Verge.io system that has been configured to accept incoming syncs, some of these rules may already be in place.  

 

Core Network  Rule - 14201 PAT Rule

Note: For systems already configured for accepting incoming syncs, this Rule may already be in place.

  • Name (required): ex: “vSAN PAT”
  • Action: Translate
  • Protocol: TCP
  • Direction: Incoming
  • Source: Any / None (default)
  • Destination Type: Custom *Custom Filter: ui Destination Ports: 14201
  • Target Type: IP/Custom   *Target IP: ui 

 *  ui is a Verge.io keyword; it must be entered in lower case, exactly as noted.

Core Network SNAT

  • Name (required): ex: “vSAN SNAT for Clone Utility”
  • Action: Translate
  • Protocol: TCP
  • Direction: Outgoing
  • Interface: Router 
  • Source: Any / None (default)
  • Destination Type: My Network Address  Destination Ports/Ranges: 14201
  • Target Type: My Router IP

 

The new rules will appear in the Rules list for the Core network.

Click Apply Rules on the left menu to put the new rules into effect.

External Network - 14201 PAT rule

  • Name (required): ex: “vsan PAT”
  • Action: Translate
  • Protocol: TCP
  • Direction: Incoming
  • Source: Source IP/IP range from which clone transfer will come. Note: It is important to Include a source IP/Network here in order to restrict incoming 14201 traffic to only valid sync/clone transfer sources. 
  • Destination Type: My Router IP   Destination Ports/Ranges: 14201
  • Target Type: IP/Custom   Target IP: ui  (This is a Verge.io keyword; it must be entered in lower case, exactly as noted)

 

The new External vsan Rule will appear in the Rules list for the External network.

Click Apply Rules on the left menu to put the new rule into effect.
