What are the requirements for encrypting the vSAN?

  • Is my data encrypted at Rest? How can I verify if my system is using encryption for the vSAN?
    • You can confirm this by navigating to Nodes>Node 1> Drives and then double-clicking on the first drive in the list.  There is a checkbox for encryption.  If it is encrypted, it will be checked.
  • My drives are not encrypted, can I change that after the system has been installed?
    • Unfortunately, encryption for the vSAN is configured during the initial installation only
  • My system has encryption configured, how does the Verge.io system start on an encrypted system?
  1. The most common method is by having encryption keys written to a usb drive during the initial installation.  In this scenario,  these drives are typically plugged in to the first two nodes of an encrypted system to boot normally.  All other nodes do not require them,  as node 1 and node 2 are the controller nodes.  The usb drive does not require much storage at all, less than 1GB.
  2. If the controller nodes do not have usb encryption keys connected, the Verge.io system will prompt an operator to type the proper encryption password to complete the power up off the Verge.io system.
  • Is my data encrypted in flight?  Is my data encrypted through the site-sync transfer?
    • Yes, by default encryption is set for all snapshot synchronizations through a site-sync.  Information about encrypting a Site Synchronization can be found in the inline help within the category titled, Site Syncs, under the section labeled 'Creating Outgoing Sync.'
